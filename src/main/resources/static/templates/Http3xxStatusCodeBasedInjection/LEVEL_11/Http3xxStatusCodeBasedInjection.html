This level demonstrates a secure redirect implementation designed to reduce phishing risk.

Unlike the vulnerable levels, this implementation only allows returnTo values that point to approved internal locations. Untrusted external redirect targets are blocked instead of being followed.

Try using trusted internal paths such as: / or /VulnerableApp/

If you provide an untrusted destination, the application should block the redirect and explain why it was rejected.

A secure redirect should not rely only on warning pages or user judgment. It should enforce an allowlist so attacker-controlled phishing destinations cannot be used in the first place.
