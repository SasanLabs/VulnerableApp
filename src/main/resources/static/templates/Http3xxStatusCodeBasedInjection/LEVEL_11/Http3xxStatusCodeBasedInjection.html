This level demonstrates a secure redirect implementation designed to reduce phishing risk.

Unlike the vulnerable levels, this implementation only allows returnTo values that point to approved internal locations. All other destinations are blocked.

! Redirect Blocked
How this secure level works: Only approved internal destinations are allowed. All other redirect targets are blocked, including external sites and unapproved internal paths.

Try your own values as well, such as: https://www.youtube.com/, https://www.netflix.com/browse, or other paths not shown below. If they are not on the approved allowlist, they should be blocked.
Quick examples:
Allowed: /VulnerableApp/ Blocked: https://google.com Blocked: /VulnerableApp/phishing/fake-login.html
A secure redirect should not rely only on warning pages or user judgment. It should enforce an allowlist so attacker-controlled phishing destinations cannot be used in the first place.
