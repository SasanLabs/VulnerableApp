This level demonstrates how Open Redirect can be used to carry out a phishing attack that harvests login credentials.

An attacker crafts a trusted-looking URL on this application with a returnTo parameter pointing to a fake login page they control. A warning popup is shown before the redirect — but many users click Continue without carefully checking the destination URL.



Clicking the button simulates opening a post-login redirect link. A warning popup will appear showing the redirect destination. Click Continue to land on the fake login page and see how credentials can be harvested.
A warning interstitial alone is not sufficient protection. As long as the server redirects to any user-supplied URL, an attacker can craft a convincing phishing page and rely on users trusting the source domain.
