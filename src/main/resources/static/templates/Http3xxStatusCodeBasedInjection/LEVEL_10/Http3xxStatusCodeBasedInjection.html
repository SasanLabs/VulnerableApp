This level demonstrates how Open Redirect can be used to carry out a phishing attack that harvests login credentials.

An attacker crafts a trusted-looking URL on this application with a returnTo parameter pointing to a fake login page they control. The server shows a warning before redirecting — but many users click Continue without carefully checking the destination URL.
